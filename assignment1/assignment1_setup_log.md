## Notes From Looking Around

W3Schools looks beginner-friendly because the page is organized into a left-side tutorial menu, short topic sections, and practice/exercise links. I did not study the Python or HTML material yet; I only skimmed the layout and noticed how the lessons are broken into small steps.

For the AI coding environment, Cursor looks like a code editor with agent-style workflows, download/get-started links, and sections for tools like agents, tab completion, review, CLI, and cloud agents. Claude Code looks more terminal-focused and says it can work with codebases, command-line tools, editors, and browser/desktop access. What confused me most was that "AI assistant" and "AI coding environment" overlap, but they are not exactly the same: a chat assistant mainly answers questions, while a coding agent can inspect files, edit code, run commands, and help with GitHub-style workflows.

The answer to my website question was that the browser does several steps after I type a URL: it resolves the domain name with DNS, makes a secure connection to the server, requests the page, downloads HTML/CSS/JavaScript/images, and then renders everything visually on the screen. One thing that surprised me was how many hidden systems are involved before a web page appears, even though it feels instant as a user. Another surprise was that modern AI coding tools are not just autocomplete; they can act more like a partner that understands the project structure and can plan changes across multiple files.
